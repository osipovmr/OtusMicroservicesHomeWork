Метрика http_server_requests_seconds_bucket представляет собой гистограмму,
которая измеряет время обработки HTTP-запросов на сервере.
Она позволяет отслеживать распределение времени обработки запросов по разным интервалам значений.

Метрика состоит из нескольких бакетов, каждый из которых представляет временной интервал.
Значения для каждого бакета показывают, сколько запросов было обработано за время, равное или большее, чем значение данного бакета.

Например, если у нас есть следующие значения для метрики http_server_requests_seconds_bucket:

http_server_requests_seconds_bucket{le="0.1"}: 100
http_server_requests_seconds_bucket{le="0.5"}: 150
http_server_requests_seconds_bucket{le="1.0"}: 200
http_server_requests_seconds_bucket{le="+Inf"}: 250
Это означает, что:

100 запросов были обработаны за 0.1 секунды или меньше
150 запросов были обработаны за 0.5 секунды или меньше
200 запросов были обработаны за 1.0 секунду или меньше
Всего было обработано 250 запросов (включая все временные интервалы)
Метрика http_server_requests_seconds_bucket обычно используется для анализа производительности сервера
и выявления проблемных запросов, которые требуют длительного времени обработки.
Также она позволяет определить процентиль времени обработки запросов,
что полезно для понимания поведения сервера при различных нагрузках.
